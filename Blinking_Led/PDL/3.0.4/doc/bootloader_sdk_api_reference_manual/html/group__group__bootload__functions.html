◆ Cy_Bootload_DoBootload()

cy_en_bootload_status_t Cy_Bootload_DoBootload ( uint32_t *  state,
uint32_t  timeout 
)

This function starts the application download and install operations.

It calls Cy_Bootload_Init() and then continually calls Cy_Bootload_Continue(). It does not return until the bootload operation is complete or an error is detected. This function provides an example of how to use the Bootloader SDK to quickly design a bootload system.

Note
This function blocks, suspending all other tasks until the bootload operation is complete. To execute other tasks while bootloading, design your code to call Cy_Bootload_Init() and Cy_Bootload_Continue() directly. Only one bootloading operation can be done at a time - the user's code must ensure this.
The timeout for a communication interface is 20 ms - enough for UART (at 115200 bps), I2C, SPI, USB-HID, and BLE OTA in most cases.
The function enables global interrupts.
Parameters
stateThe pointer to a bootloading state variable. See Bootload State.
timeoutThe timeout in milliseconds for bootloading.
Returns
See cy_en_bootload_status_t.

This function starts the application download and install operations.

Make subsequent calls to Cy_Bootload_Continue() to continue the process.

Only one bootloading operation can be done at a time - the user's code must ensure this.

Parameters
stateThe pointer to a bootloading state variable See Bootload State
paramsThe pointer to a bootloader parameters structure See cy_stc_bootload_params_t
Returns
See cy_en_bootload_status_t.
  • CY_BOOTLOAD_SUCCESS if successful.
  • CY_BOOTLOAD_ERROR_UNKNOWN if a NULL pointer is passed.

◆ Cy_Bootload_DataChecksum()

uint32_t Cy_Bootload_DataChecksum ( const uint8_t *  address,
uint32_t  length,
cy_stc_bootload_params_t *  params 
)

This function computes a CRC-32C for the provided number of bytes contained in the provided buffer.


This function is used to validate the Program Data and Verify Data bootloader commands and a metadata row.

Note
Ensure the Crypto block is properly initialized if CY_BOOTLOAD_OPT_CRYPTO_HW is set.
Parameters
addressThe pointer to a buffer containing the data to compute the checksum for.
lengthThe number of bytes in the buffer to compute the checksum for.
paramsThe pointer to a bootloader parameters structure. See cy_stc_bootload_params_t .
Returns
CRC-32C for the provided data.

◆ Cy_Bootload_GetAppMetadata()

cy_en_bootload_status_t Cy_Bootload_GetAppMetadata ( uint32_t  appId,
uint32_t *  verifyAddress,
uint32_t *  verifySize 
)

Reads application metadata to verifyAddress and verifySize.

The metadata is supposed to be located in internal Flash memory.

This is a weak function and the user may override it in the user's code by providing a function with the same name. This allows the user to place metadata in any NVM.

Note
It is assumed appId is a valid application number.
Parameters
appIdThe application number.
verifyAddressThe pointer to a variable where an application verified area start address is stored.
verifySizeThe pointer to a variable where a size of verified application area is stored.
Returns
See cy_en_bootload_status_t.

This is a helper function for Cy_Bootload_Continue().

This function sets application metadata and updates a metadata checksum.

This function uses params->dataBuffer for the read and write NVM.

This function copies an application from a temporary location in Flash to its destination location in Flash.

This function is typically called when updating an application used as part of an update process, for example updating a BLE stack.

rowSizeThe size of a Flash row in bytes.
paramsThe pointer to a bootloader parameters structure. See cy_stc_bootload_params_t .
Returns
See cy_en_bootload_status_t.

◆ Cy_Bootload_ReadData()

cy_en_bootload_status_t Cy_Bootload_ReadData ( uint32_t  address,
uint32_t  length,
uint32_t  ctl,
cy_stc_bootload_params_t *  params 
)

This function must be implemented in the user's code.

Reads buffer from Flash, SMIF, or any other external memory type with custom pre and post read commands.

Parameters
addressThe address from where to read data, must be aligned to a Flash row, SMIF page, etc.
lengthThe length in bytes of data to read, must be multiple of a Flash row, SMIF page, etc.
ctlAdditional features of the read function:
  • CY_BOOTLOAD_IOCTL_READ - Only read.
  • CY_BOOTLOAD_IOCTL_COMPARE - Compare the data in the buffer with the data in memory.
  • CY_BOOTLOAD_IOCTL_BHP - Decrypt data before comparing the buffer with memory, if the bootloader Host provided encrypted data.
paramsThe pointer to a bootloader parameters structure. See cy_stc_bootload_params_t .
Returns
See cy_en_bootload_status_t
  • CY_BOOTLOAD_SUCCESS - If successful.
  • CY_BOOTLOAD_ERROR_LENGTH if The length value is invalid.
  • CY_BOOTLOAD_ERROR_ADDRESS if The address is invalid.

This function must be implemented in the user's code.

Writes the buffer to Flash, SMIF, or any other external memory type with custom pre and post write commands.

Parameters
addressThe address to write data to, must be aligned to a Flash row, SMIF page, etc.
lengthThe length in bytes of data to be written, must be multiple of a Flash row, SMIF page, etc.
ctlAdditional features of the write function:
  • CY_BOOTLOAD_IOCTL_WRITE - Only write.
  • CY_BOOTLOAD_IOCTL_ERASE - Erase the sector, the sector size can be bigger than the size of the page to write.
  • CY_BOOTLOAD_IOCTL_BHP - Decrypt data before writing to memory, if the Bootloader Host provided encrypted data.
paramsThe pointer to a bootloader parameters structure. See cy_stc_bootload_params_t .
Returns
See cy_en_bootload_status_t.
  • CY_BOOTLOAD_SUCCESS - If successful.
  • CY_BOOTLOAD_ERROR_LENGTH if The length value is invalid.
  • CY_BOOTLOAD_ERROR_ADDRESS if The address is invalid.

This function receives a command packet from the Bootloader Host via the communication channel. The function waits for a timeout until all bytes are received.

sizeThe number of bytes to read.
countThe pointer to the variable that contains the number of received bytes.
timeoutThe time to wait before the function returns because of a timeout, in milliseconds.
Returns
The status of the transmit operation:

  • CY_BOOTLOAD_ERROR_TIMEOUT - If no data is received.
  • See cy_en_bootload_status_t.
