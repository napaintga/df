Macro Definition Documentation

◆ CY_BOOTLOAD_SILICON_ID

#define CY_BOOTLOAD_SILICON_ID   (0u)

The silicon ID for the chip value used by the Bootloader SDK.

If set to ZERO Bootloader Host, is supposed to ignore it. The Enter bootloader command uses this to respond to a Bootloader Host.

◆ CY_BOOTLOAD_SILICON_REV

#define CY_BOOTLOAD_SILICON_REV   (0u)

Silicon Revision for the chip.

This parameter is an addition to CY_BOOTLOAD_SILICON_ID.

◆ CY_BOOTLOAD_GOLDEN_IMAGE_IDS

#define CY_BOOTLOAD_GOLDEN_IMAGE_IDS ( )    0u

List of Golden Image Application IDs.

Here "Golden Image Application" means an application that cannot be changed with CommandProgramData()

Usage. Define the list of Golden Image Application IDs without enclosing parenthesis, e.g.

#define CY_BOOTLOAD_GOLDEN_IMAGE_IDS() 0u, 1u, 3u

later it is used in cy_bootload.c file:

uint8_t goldenImages[] = { CY_BOOTLOAD_GOLDEN_IMAGE_IDS() };

◆ CY_BOOTLOAD_MAX_APPS

#define CY_BOOTLOAD_MAX_APPS   (2u)

The number of applications in the metadata, for 512 bytes in a Flash row - 63 is the maximum possible value, because 4 bytes are reserved for the entire metadata CRC.

The smallest metadata size if CY_BOOTLOAD_MAX_APPS * 8 (bytes per one app) + 4 (bytes for CRC-32C)

◆ CY_BOOTLOAD_OPT_SEND_DATA

#define CY_BOOTLOAD_OPT_SEND_DATA   (1)

A non-zero value enables the Send Data bootloader command.

If the "Send Data" bootloader command is enabled, packetBuffer and dataBuffer must be non-overlapping.

Else, dataBuffer must be inside packetBuffer with an offset of CY_BOOTLOAD_PACKET_DATA_IDX, typically 4 bytes.
params->dataBuffer = &packetBuffer[4];

Note
that packetBuffer in this case must be 4 bytes aligned, as dataBuffer is required to be 4 bytes aligned.
