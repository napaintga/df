cy_en_ble_eco_status_t Cy_BLE_EcoStart ( const cy_stc_ble_bless_eco_cfg_params_t *  ecoConfig)

Configures and starts BLE ECO clock.

If Cy_BLE_Start()/Cy_BLE_Enable() APIs are called by the application, the stack enables the BLE ECO clock automatically with the following default parameters:

Parameter Value
ECO Frequency CY_BLE_DEFAULT_ECO_FREQ
Divider CY_BLE_DEFAULT_ECO_DIV
Startup time CY_BLE_DEFAULT_OSC_STARTUP_DELAY_LF
Load cap CY_BLE_DEFAULT_CAP_TRIM_VALUE

If there is a need to start the BLE ECO with non-default parameters, call the Cy_BLE_EcoConfigure() function with the custom configuration each time before calling the Cy_BLE_Enable() function.

This clock is stopped in Deep Sleep and Hibernate power modes.

Returns
cy_en_ble_eco_status_t : The return value indicates if the function succeeded or failed. The possible error codes:
Parameters
ecoConfigClock configuration of type 'cy_stc_ble_bless_eco_cfg_params_t'.
Returns
cy_en_ble_eco_status_t : The return value indicates if the function succeeded or failed. The possible error codes:
Error Codes Description
CY_BLE_ECO_SUCCESS Function completed successfully.
CY_BLE_ECO_BAD_PARAM Wrong input parameter.
CY_BLE_ECO_RCB_CONTROL_LL RCB Control is with BLE Link Layer.
CY_BLE_ECO_ALREADY_STARTED BLE ECO clock is already started.
CY_BLE_ECO_HARDWARE_ERROR RCB or BLE ECO operation failed.
Side Effects
The I/O pins will be automatically unfrozen coming out of hibernate when the BLE ECO is in use.
Note
Limitation: Do not call this API in case if BLE is executed with LPM. There is a risk that when Cy_BLE_EcoStart() function is called on one core, the BLE is in deep-sleep mode on other. It will cause a fault hard exception.
